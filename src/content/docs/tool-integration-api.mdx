## 接入模式

<Tabs items={["纯配置工具", "通用 AI 工具", "外部服务工具", "站内复杂工具"]}>
  <Tab>
    适合只想复用平台账号体系、后台配置和入口管理的工具。

    推荐做法：

    - 导入工具定义
    - 配好字段
    - 入口指向外部地址或内部页面
  </Tab>
  <Tab>
    适合主要依赖统一 AI 网关的工具。

    推荐做法：

    - 导入工具定义
    - 配置 `features`
    - 在 `/admin/ai` 绑定模型和 provider
  </Tab>
  <Tab>
    适合前后端独立部署的产品。

    推荐做法：

    - `authMode = launch_ticket`
    - 平台生成启动票据
    - 外部服务端用 `runtime token` 换用户身份并读取配置
  </Tab>
  <Tab>
    适合像 `RedInk` 这样有专属页面、专属流程和专属适配层的工具。

    推荐做法：

    - 工具业务逻辑放工具模块
    - 用户、AI、积分、存储仍走平台统一能力
  </Tab>
</Tabs>

## 标准接入流程

<Steps>
  <Step>
    管理员进入 `/zh/admin/tool-config`，导入工具定义 JSON
  </Step>
  <Step>
    在 `/zh/admin/tool-config` 填管理员默认配置、密钥和 JSON 字段
  </Step>
  <Step>
    如果工具需要 AI，去 `/zh/admin/ai` 给 `feature` 绑定模型和 provider
  </Step>
  <Step>
    如果工具要上传资源，去 `/zh/admin/storage` 检查对象存储和前缀规则
  </Step>
  <Step>
    如果工具允许用户个人覆盖配置，到 `/zh/dashboard/settings` 验证字段是否按预期暴露
  </Step>
  <Step>
    如果工具是外部服务，再串联 `launch_ticket`、`session/exchange`、`runtime`、`runtime-save`
  </Step>
</Steps>

## 常用平台接口

### 1. 会话与账户

#### `GET /api/platform/session`

返回当前登录用户、套餐和积分余额。

```json
{
  "success": true,
  "user": {
    "id": "user_xxx",
    "name": "Alice",
    "email": "alice@example.com",
    "image": null,
    "role": "user"
  },
  "plan": {
    "code": "pro",
    "name": "Pro",
    "hasActiveSubscription": true
  },
  "credits": {
    "balance": 520,
    "status": "active"
  }
}
```

### 2. 积分

#### `POST /api/platform/credits/check`

```json
{
  "amount": 3
}
```

```json

  "description": "生成会议纪要",
  "metadata": {
    "tool": "notes-ai",
```

## 外部工具接入协议

<Callout type="idea" title="外部工具的核心原则">
  浏览器不应该直接持有 `runtime token`。正确做法是：平台前端拿一次性 `ticket`，外部工具服务端持有 `runtime token` 去换用户身份和读写配置。
</Callout>

### 启动时序

<Steps>
  <Step>
    平台前端调用 `GET /api/platform/tools/{toolKey}/launch`
  </Step>
  <Step>
    平台拿到 `ticket` 和 `launchUrl` 后跳转到外部工具
  </Step>
  <Step>
    外部工具服务端用 `POST /api/platform/tools/session/exchange` 换到当前用户
  </Step>
  <Step>
    外部工具服务端再调用 `runtime` / `runtime-save` 读取和写回工具配置
  </Step>
</Steps>

### `GET /api/platform/tools/{toolKey}/launch`

成功响应示例：

```json
{
  "success": true,
  "tool": "notes-ai",
  "ticket": "ticket_xxx",
  "launchUrl": "https://notes.tripai.icu?ticket=ticket_xxx&projectKey=nextdevtpl",
  "expiresAt": "2026-04-17T10:30:00.000Z"
}
```

### `POST /api/platform/tools/session/exchange`

请求头：

```http
Authorization: Bearer <runtime-token>
```

请求体：

```json
{
  "projectKey": "nextdevtpl",
  "tool": "notes-ai",
  "ticket": "ticket_xxx"
}
```

成功响应：

```json
{
  "success": true,
  "toolKey": "notes-ai",
  "user": {
    "id": "user_xxx",
    "name": "Alice",
    "email": "alice@example.com"
  },
  "expiresAt": "2026-04-17T10:35:00.000Z"
}
```

### `POST /api/platform/tool-config/runtime`

请求头：

```http
Authorization: Bearer <runtime-token>
```

请求体：

```json
{
  "projectKey": "nextdevtpl",
  "tool": "notes-ai",
  "userId": "user_xxx",
  "knownRevision": 12
}
```

### `POST /api/platform/tool-config/runtime-save`

请求头：

```http
Authorization: Bearer <runtime-token>
```

请求体：

```json
{
  "projectKey": "nextdevtpl",
  "tool": "notes-ai",
  "userId": "user_xxx",
  "values": {
    "config2": "friendly"
  },
  "clearSecrets": []
}
```

### `GET /api/platform/tool-config/revision?projectKey=nextdevtpl&tool=notes-ai`

请求头：

```http
Authorization: Bearer <runtime-token>
```

用途：

- 只读取版本号
- 适合检测配置是否变化

## 推荐联调顺序

<Steps>
  <Step>
    先完成工具定义导入，不要先写接口调用
  </Step>
  <Step>
    先把后台默认配置、密钥和 JSON 字段填完整
  </Step>
  <Step>
    再绑定 AI provider 和模型
  </Step>
  <Step>
    再验证上传和对象存储前缀
  </Step>
  <Step>
    再打通外部票据和运行时配置
  </Step>
  <Step>
    最后串联完整业务流：AI、积分、归档、错误处理
  </Step>
</Steps>

## 常见失败原因

<Tabs items={["导入后不能用", "用户看不到配置", "外部工具拿不到用户", "上传路径不对"]}>
  <Tab>

<ul>
  <li>只导入了工具定义，没有补后台默认配置</li>
  <li>AI 功能没有在 <code>/admin/ai</code> 绑定模型</li>
  <li>工具被停用</li>
</ul>

  </Tab>
  <Tab>

<ul>
  <li>字段被设置成 <code>adminOnly = true</code></li>
  <li>没开 <code>userOverridable = true</code></li>
</ul>

  </Tab>
  <Tab>

<ul>
  <li><code>authMode</code> 不是 <code>launch_ticket</code></li>
  <li><code>runtime token</code> 没有 <code>session:exchange</code> 权限</li>
  <li>启动票据过期或已被消费</li>
</ul>

  </Tab>
  <Tab>

<ul>
  <li><code>toolKey</code> 传错</li>
  <li><code>purpose</code> 传错</li>
  <li>后台没有对应的工具存储规则</li>
</ul>

  </Tab>
</Tabs>

## 一句话结论

<Callout type="success" title="推荐做法">
  新工具接入时，先导入工具定义，再补管理员配置，再绑定 AI 和存储，最后按是否为外部服务决定是否接 `launch_ticket` 和运行时接口。不要反过来做。
</Callout>
