# RedInk 用户可见模型目录开发计划

## 目标

为 `redink` 增加按场景选择模型的能力：

- 标题与文案可选择文本模型
- 图片生成可选择图片模型
- 用户只看到管理员开放的模型子集和展示别名
- 管理员配置来源于 `NextDevTpl` 的工具配置
- 实际可选项必须是 AI 网关模型绑定的子集

## 配置原则

1. AI 网关模型绑定继续作为平台真实可用模型全集
2. `redink` 用户可见模型目录作为工具配置中的管理员专用字段
3. 用户不可在 `NextDevTpl` 设置页查看或修改该目录
4. `redink` 前端只展示别名和说明，实际提交 `modelKey`
5. AI 网关继续做最终校验，保证模型白名单和能力声明生效

## 阶段拆分

### Phase 1

- 在 `NextDevTpl` 增加 `redink` 的管理员模型目录配置
- 增加用户态模型目录接口
- 接口按 AI 网关绑定、模型白名单和能力声明过滤

状态：已完成

### Phase 2

- `redink` 增加平台代理接口
- `redink` 前端拉取并缓存模型目录

状态：已完成

### Phase 3

- 文本生成链路支持用户选择模型
- 覆盖标题文案、商品文案、商品发布文案

状态：未开始

### Phase 4

- 图片生成链路支持用户选择模型
- 覆盖商品发布图和通用图片生成

状态：未开始

### Phase 5

- 回归测试
- 文档收尾
- 最终联调

状态：未开始

## 当前进度

- 已完成 Phase 1：管理员可配置 `redink` 用户可见模型目录，用户态目录接口会按白名单和能力声明过滤
- 已完成 Phase 2：新增 `redink/request-result` 代理结果接口，`model-options` 已支持 `ETag` 与 `304`，前端可按 revision 与缓存头复用目录
- Phase 3 将补齐文本生成代理，覆盖标题、正文和商品发布文案的用户选模入口
